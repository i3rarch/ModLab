# ModLab: Программный модуль для стенда «Методы цифровой модуляции»

## О проекте

**ModLab** — это программное обеспечение для управления учебно-исследовательским стендом, предназначенным для изучения методов цифровой модуляции. Приложение предоставляет пользовательский интерфейс для взаимодействия с передатчиком (ПРД), приемником (ПРМ) и генератором помех, позволяя в реальном времени изменять параметры радиоканала и анализировать результаты.

Проект разработан с использованием фреймворка **Qt 6** на языке **C++** и предназначен для сборки в среде **Visual Studio**.

## Основные возможности

- **Конфигурация общих параметров:** Установка несущей частоты, типа модуляции (2-FSK, GFSK, ASK/OOK, MSK), скорости передачи и девиации частоты.
- **Управление передатчиком (ПРД):**
    - Регулировка выходной мощности.
    - Выбор режима пакетов (фиксированная или переменная длина).
    - Выбор источника данных: текстовое поле, счетчик или файл.
    - Управление процессом передачи.
- **Эмуляция канала связи:**
    - Управление генератором помех (белый шум, синусоидальная помеха).
    - Регулировка мощности помехи.
- **Анализ данных на приемнике (ПРМ):**
    - Отображение принятых данных.
    - Мониторинг ключевых показателей качества связи: RSSI, LQI.
    - Подсчет принятых пакетов и пакетов с ошибками (CRC).
    - Расчет коэффициента ошибок по пакетам (PER).

> Все действия пользователя и ключевые события системы записываются в лог с возможностью очистки и сохранения в файл.

## Сборка и запуск

### Требования:
- **Visual Studio** (2019 или новее) с компилятором MSVC.
- **Qt 6** (рекомендуется версия 6.2 и выше) с установленными модулями `Widgets` и `SerialPort`.
- **Qt VS Tools** для интеграции с Visual Studio.

### Процесс сборки:
1. Откройте файл проекта `.sln` в Visual Studio.
2. Убедитесь, что в настройках проекта (`Свойства проекта -> Qt Project Settings`) правильно указан путь к установленной версии Qt.
3. В `Qt Modules` должны быть подключены `Core`, `Gui`, `Widgets` и `SerialPort`.
4. В настройках компилятора (`C/C++ -> Командная строка`) должен быть установлен флаг `/utf-8` для корректной обработки кириллицы в исходном коде.
5. Выполните сборку проекта (клавиша F7 или меню `Сборка -> Собрать решение`).
6. Запустите приложение (клавиша F5).

## Структура интерфейса

Главное окно приложения разделено на несколько функциональных блоков:

1.  **Подключение устройств:** Выбор COM-портов для ПРД, ПРМ и генератора помех.
2.  **Общие параметры системы:** Настройка несущей частоты, модуляции и скорости передачи.
3.  **Передатчик (ПРД):** Управление параметрами и данными для передачи.
4.  **Канал связи / Помехи:** Управление генератором помех.
5.  **Приемник (ПРМ):** Отображение принятых данных и метрик качества связи.
6.  **Логгирование:** Журнал всех событий в системе.
